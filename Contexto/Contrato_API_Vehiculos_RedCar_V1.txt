 CONTRATO API Microservicio de Vehículos – RedCar v1.0.0 | Equipo de Catálogo de Vehículos Abril 2026 1. Información General Campo Valor Nombre de la API API de Vehículos – RedCar Versión v1.0.0 Estado Producción Dueño Equipo de Catálogo de Vehículos Endpoint base /api/v1 Métodos HTTP GET, POST, PATCH Formato de respuesta JSON Estrategia de integración Consumo síncrono (REST/HTTP) Autenticación Bearer Token (Authorization header) Descripción Permite consultar vehículos, localizaciones, categorías, extras, y gestionar reservas desde el canal de booking externo. Nota: El campo origen_canal_reserva se usa para identificar que la reserva provino del canal de booking externo. Siempre enviar el valor "BOOKING" en ese campo al crear una reserva. 2. Endpoints GET — Consulta y Búsqueda Estos 7 endpoints permiten al equipo de booking consultar disponibilidad, catálogos y detalles antes de que el cliente confirme una reserva. Endpoints Externos — Gestión de Reservas Estos 4 endpoints permiten al equipo de booking crear y gestionar reservas en el sistema de RedCar. Son los contratos que se activan cuando un cliente confirma, consulta o cancela una reserva desde la plataforma de booking. Endpoint 8: Crear reserva Atributo Detalle Método HTTP POST URL /api/v1/reservas Descripción Crea una nueva reserva en el sistema RedCar. Booking envía todos los datos del cliente, vehículo, fechas, conductores y extras seleccionados. RedCar devuelve el código de reserva generado. Booking invoca este endpoint cuando el cliente solicita la cancelación desde la plataforma de booking. RedCar cambia el estado a CAN y registra el motivo. Parámetros de entrada Parámetro Tipo Requerido Descripción codigoReserva string Sí Código único de reserva.
