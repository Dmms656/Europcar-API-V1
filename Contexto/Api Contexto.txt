DbContext sugerido

Tu contexto debería verse conceptualmente así:

public class RentalDbContext : DbContext
{
    public RentalDbContext(DbContextOptions<RentalDbContext> options) : base(options) { }

    public DbSet<ClienteEntity> Clientes => Set<ClienteEntity>();
    public DbSet<ConductorEntity> Conductores => Set<ConductorEntity>();
    public DbSet<VehiculoEntity> Vehiculos => Set<VehiculoEntity>();
    public DbSet<ReservaEntity> Reservas => Set<ReservaEntity>();
    public DbSet<UsuarioAppEntity> UsuariosApp => Set<UsuarioAppEntity>();
    public DbSet<RolEntity> Roles => Set<RolEntity>();
    public DbSet<UsuarioRolEntity> UsuariosRoles => Set<UsuarioRolEntity>();
    public DbSet<AudEventoEntity> AuditoriaEventos => Set<AudEventoEntity>();

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        modelBuilder.HasDefaultSchema("public");

        modelBuilder.ApplyConfigurationsFromAssembly(typeof(RentalDbContext).Assembly);

        base.OnModelCreating(modelBuilder);
    }
}
Diferencia importante con PostgreSQL

En SQL Server el documento usa rowversion e IsRowVersion().
En PostgreSQL eso no existe igual. En tu caso conviene manejar concurrencia con:

columna numérica row_version
o token de concurrencia configurado con EF Core

No debes copiar literalmente esa parte del documento.

Qué debes cambiar del documento para que quede bien con PostgreSQL
1. Proveedor de EF Core

Donde el documento usa SQL Server, debes usar PostgreSQL.

Paquetes recomendados:

Microsoft.EntityFrameworkCore
Npgsql.EntityFrameworkCore.PostgreSQL
Microsoft.EntityFrameworkCore.Design
Microsoft.EntityFrameworkCore.Tools
Microsoft.AspNetCore.Authentication.JwtBearer
Asp.Versioning.Mvc
Asp.Versioning.Mvc.ApiExplorer
Swashbuckle.AspNetCore
2. Connection string

En vez de la del ejemplo con UseSqlServer, tu API debe usar UseNpgsql. El documento muestra que Program.cs debe centralizar DI y DbContext.

Ejemplo:

{
  "ConnectionStrings": {
    "RentalDb": "Host=localhost;Port=5432;Database=europcar_db;Username=postgres;Password=admin"
  }
}

Y en DI:

services.AddDbContext<RentalDbContext>(options =>
    options.UseNpgsql(configuration.GetConnectionString("RentalDb")));

El documento usa crm.
Tú debes mapear:

rental.clientes

rental.reservas

4. Autenticación

El archivo separa muy bien usuarios de aplicación y seguridad de base, y eso te conviene conservar: los usuarios de PostgreSQL no son los usuarios funcionales de la API; la API debe autenticarse contra security.usuarios_app y roles relacionados.
